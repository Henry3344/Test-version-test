Privacy & Security Policy

FlashDrop Pro is built with a single principle: your data never belongs to us. This document explains exactly how FlashDrop Pro handles privacy, security, and data protection — in plain, verifiable terms.

1. Pure Peer-to-Peer Architecture

FlashDrop Pro uses WebRTC-based peer-to-peer (P2P) connections. Files and text are transferred directly between browsers.

  • No file data is uploaded to or stored on our servers
  • No cloud storage, no intermediaries
  • Once the browser tab is closed, all session data is destroyed

2. Zero Server Storage

FlashDrop Pro does not store:

  • Files or file fragments / Text content
  • Download history / Transfer metadata

Our servers are used only for minimal connection signaling — never for content delivery.

3. Zero Logs Policy

  • No IP addresses / No User identities
  • No File names or sizes / No Transfer timestamps

4. End-to-End Session Control

Users define strict access limits: One-time download, limited counts, or manual stop. Once reached, the session self-destructs automatically.

5. Encryption & Transport Security

All transfers occur over encrypted WebRTC channels using modern standards. Data is encrypted in transit and never decrypted by our servers.

隐私与安全说明

FlashDrop Pro 的核心原则只有一个：你的数据永远不属于我们。

本页面将以清晰、可验证的方式说明 FlashDrop Pro 如何处理隐私、安全与数据保护。

1. 纯 P2P 架构

基于 WebRTC 实现浏览器之间的点对点连接，文件和文本直接在设备之间传输。

  • 文件数据永不上传或存储在服务器上
  • 没有云端中转，没有中间商
  • 一旦关闭浏览器标签，所有会话数据立即销毁

2. 零服务器存储

  • 不存储文件/不存储文本
  • 不存储下载记录/不存储传输元数据

3. 零日志政策

  • 不记录 IP / 不记录用户身份
  • 不记录文件名或大小 / 不记录传输时间戳

4. 会话自毁机制

每次传输都是临时会话，达到下载限制或关闭页面后即自动销毁。

5. 加密传输

所有数据通过 WebRTC 加密通道传输，服务器无法解密或查看内容。

プライバシーとセキュリティポリシー

FlashDrop Proは、「ユーザーのデータはユーザーだけのもの」というシンプルな原則に基づいて構築されています。この文書では、FlashDrop Proがプライバシー、セキュリティ、およびデータ保護をどのように扱っているかを説明します。

1. 純粋な P2P アーキテクチャ

WebRTCを使用したブラウザ間の直接接続により、ファイルとテキストはデバイス間で直接転送されます。

  • ファイルデータがサーバーにアップロードまたは保存されることはありません
  • クラウドストレージや仲介者は存在しません
  • タブを閉じると、すべてのセッションデータは即座に破棄されます

2. サーバー保存ゼロ

  • ファイルやテキストの内容を保存しません
  • ダウンロード履歴や転送メタデータを保存しません

3. ログ取得ゼロ

  • IPアドレスやユーザー識別情報を記録しません
  • ファイル名やサイズの記録も行いません

4. エンドツーエンドのセッション管理

1回限りのダウンロードや回数制限など、厳格なアクセス制限を設定できます。制限に達すると、セッションは自動的に自己破壊されます。

5. 暗号化と転送の安全性

すべての転送は、最新のブラウザ標準に従い、暗号化されたWebRTCチャネルを介して行われます。

개인정보 및 보안 정책

FlashDrop Pro의 핵심 원칙은 단 하나입니다: 사용자의 데이터는 절대 저장되지 않습니다. 본 문서는 FlashDrop Pro가 개인정보, 보안 및 데이터 보호를 처리하는 방식을 명확하게 설명합니다.

1. 순수 P2P 아키텍처

WebRTC를 기반으로 브라우저 간 직접 연결(P2P)을 실현하며, 파일과 텍스트는 장치 간 직접 전송됩니다.

  • 파일 데이터가 서버에 업로드되거나 저장되지 않습니다
  • 클라우드 스토리지나 중개 서버가 없습니다
  • 페이지를 닫으면 모든 세션 데이터가 즉시 삭제됩니다

2. 서버 저장소 제로

  • 파일 및 텍스트 콘텐츠를 저장하지 않습니다
  • 다운로드 기록 및 전송 메타데이터를 저장하지 않습니다

3. 무로그 정책

  • IP 주소 및 사용자 신원을 기록하지 않습니다
  • 파일명 및 크기를 기록하지 않습니다

4. 종단 간 세션 제어

1회 다운로드 또는 다운로드 횟수 제한을 설정할 수 있습니다. 제한에 도달하면 세션은 자동으로 자가 파괴됩니다.

Política de Privacidad y Seguridad

FlashDrop Pro se basa en un principio único: sus datos nunca nos pertenecen. Este documento explica cómo manejamos la privacidad y la protección de datos.

1. Arquitectura P2P Pura

Utilizamos conexiones directas WebRTC. Los archivos y textos se transfieren directamente entre navegadores.

  • Los datos nunca se cargan ni se almacenan en nuestros servidores
  • Sin almacenamiento en la nube ni intermediarios
  • Al cerrar la pestaña, todos los datos se destruyen

2. Almacenamiento Cero en el Servidor

  • No guardamos archivos ni contenido de texto
  • No guardamos historial de descargas ni metadatos de transferencia

3. Política de Cero Registros (Logs)

  • No registramos direcciones IP ni identidades de usuario
  • No registramos nombres ni tamaños de archivos

4. Control de Sesión de Extremo a Extremo

Puede definir límites de acceso estrictos: descarga única o limitada. Una vez alcanzado el límite, la sesión se autodestruye automáticamente.

गोपनीयता और सुरक्षा नीति

FlashDrop Pro एक ही सिद्धांत पर बना है: आपका डेटा कभी भी हमारा नहीं होता। यह दस्तावेज़ बताता है कि हम आपकी गोपनीयता और सुरक्षा का प्रबंधन कैसे करते हैं।

1. शुद्ध P2P आर्किटेक्चर

फ़ाइलें और टेक्स्ट सीधे ब्राउज़रों के बीच (WebRTC के माध्यम से) ट्रांसफ़र किए जाते हैं।

  • डेटा हमारे सर्वर पर कभी अपलोड या स्टोर नहीं होता
  • कोई क्लाउड स्टोरेज या बिचौलिया नहीं है
  • टैब बंद होते ही सारा डेटा नष्ट हो जाता है

2. ज़ीरो सर्वर स्टोरेज

  • हम फ़ाइल सामग्री या टेक्स्ट स्टोर नहीं करते
  • कोई डाउनलोड इतिहास या मेटाडेटा स्टोर नहीं किया जाता

3. ज़ीरो लॉग्स नीति

  • हम IP एड्रेस या उपयोगकर्ता की पहचान लॉग नहीं करते
  • फ़ाइल के नाम या आकार को रिकॉर्ड नहीं किया जाता
