Privacy & Security Policy

FlashDrop Pro is built with a single principle: your data never belongs to us. This document explains exactly how FlashDrop Pro handles privacy, security, and data protection.

1. Pure Peer-to-Peer Architecture

FlashDrop Pro uses WebRTC-based peer-to-peer connections. Files and text are transferred directly between browsers.

  • No file data is uploaded or stored on servers
  • No cloud storage, no intermediaries
  • Once the browser tab is closed, all session data is destroyed

Signaling data required to establish peer-to-peer connections is temporary, minimal, and never linked to file contents.

2. Zero Server Storage

  • Files or text content
  • Download history / transfer metadata

3. Zero Logs Policy

  • We do not intentionally collect, store, or analyze IP addresses or user identities
  • Any transient network metadata required for connection establishment is not retained
  • No file names, sizes, or timestamps are recorded

4. End-to-End Session Control

Users define strict access limits: one-time download, limited counts, or manual stop. Once reached, session self-destructs automatically.

5. Encryption & Transport Security

All transfers occur over encrypted WebRTC channels using DTLS-SRTP. Data is encrypted in transit and never decrypted by our servers.

Optional client-side AES-256 encryption may be enabled before transmission. Encryption keys are generated and applied locally and are never transmitted or stored.

Transparency Notice

As of January 2026, FlashDrop Pro has never received any government, legal, or third-party requests for user data or content.

隐私与安全说明

FlashDrop Pro 的核心原则只有一个：你的数据永远不属于我们。本页面说明 FlashDrop Pro 如何处理隐私与数据安全。

1. 纯 P2P 架构

基于 WebRTC 实现浏览器间点对点连接，文件和文本直接在设备间传输。

  • 文件数据永不上传或存储在服务器上
  • 无云端存储，无中间商
  • 关闭标签页后，所有会话数据立即销毁

用于建立 P2P 连接的信令数据仅为临时存在，且不包含任何文件内容。

2. 零服务器存储

  • 文件或文本内容
  • 下载记录或传输元数据

3. 零日志政策

  • 我们不会主动收集、存储或分析 IP 地址或用户身份
  • 连接建立所需的临时网络信息不会被保留
  • 不记录文件名、大小或时间戳

4. 会话自毁机制

可设置下载次数限制或手动停止，达到限制后会话自动销毁。

5. 加密传输

所有传输通过 WebRTC 加密通道进行，服务器无法解密数据。

可选启用本地 AES-256 客户端加密，密钥仅存在于用户设备本地。

プライバシーとセキュリティポリシー

FlashDrop Proは「ユーザーデータは常にユーザーに帰属します」という原則に基づいて構築されています。

1. 純粋なP2Pアーキテクチャ

WebRTCによりブラウザ間で直接ファイル・テキストを送信します。

  • サーバーにアップロード/保存されません
  • クラウドや仲介者は使用しません
  • タブを閉じるとセッションデータは破棄されます

P2P接続確立に必要なシグナリングデータは一時的で、ファイル内容とは関連付けられません。

2. サーバー保存ゼロ

  • ファイル/テキスト内容は保存されません
  • ダウンロード履歴/転送メタデータも保存されません

3. ログゼロ

  • IPやユーザー識別情報を意図的に収集・保存しません
  • 接続に必要な一時的なネットワーク情報は保持されません
  • ファイル名/サイズ/タイムスタンプは記録されません

4. セッション管理

ダウンロード回数制限や手動停止が可能。制限到達で自動破棄。

5. 暗号化

すべての転送は WebRTC の暗号化チャネルで実施されます。

必要に応じて AES-256 によるローカル暗号化を有効化できます。

5. 암호화

모든 전송은 WebRTC 암호화 채널 사용

선택적으로 AES-256 클라이언트 측 암호화를 사용할 수 있습니다.

Política de Privacidad y Seguridad

FlashDrop Pro se basa en un principio: sus datos siempre le pertenecen a usted.

1. Arquitectura P2P Pura

Se usan conexiones WebRTC directas entre navegadores para transferir archivos y textos.

  • Datos no se suben ni almacenan en servidores
  • No hay almacenamiento en nube ni intermediarios
  • Al cerrar la pestaña, los datos de sesión se destruyen

Los datos de señalización necesarios para establecer la conexión P2P son temporales y no están vinculados al contenido.

2. Cero almacenamiento en servidor

  • No almacenamos archivos ni contenido de texto
  • No almacenamos historial de descargas ni metadatos

3. Política de cero logs

  • No recopilamos ni almacenamos intencionalmente direcciones IP o identidades de usuario
  • La información de red transitoria necesaria para la conexión no se conserva
  • No se registran nombres, tamaños ni marcas de tiempo de archivos

4. Control de sesión

Se pueden definir límites de descarga. Al alcanzarlos, la sesión se autodestruye.

5. Cifrado

Todas las transferencias usan canales WebRTC cifrados.

Opcionalmente se puede habilitar cifrado AES-256 del lado del cliente.

गोपनीयता और सुरक्षा नीति

FlashDrop Pro इस सिद्धांत पर बना है: आपका डेटा हमेशा आपका ही रहता है

1. शुद्ध P2P आर्किटेक्चर

फ़ाइल और टेक्स्ट सीधे ब्राउज़र के बीच (WebRTC) ट्रांसफर होते हैं।

  • डेटा सर्वर पर अपलोड/स्टोर नहीं होता
  • कोई क्लाउड स्टोरेज/बीचौलिया नहीं
  • टैब बंद होते ही सारा डेटा नष्ट हो जाता है

P2P कनेक्शन स्थापित करने के लिए आवश्यक सिग्नलिंग डेटा अस्थायी होता है और फ़ाइल सामग्री से जुड़ा नहीं होता।

2. ज़ीरो सर्वर स्टोरेज

  • हम फ़ाइल या टेक्स्ट स्टोर नहीं करते
  • कोई डाउनलोड हिस्ट्री या मेटाडेटा स्टोर नहीं

3. ज़ीरो लॉग्स नीति

  • हम IP पते या उपयोगकर्ता पहचान को जानबूझकर एकत्र या संग्रहित नहीं करते
  • कनेक्शन के लिए आवश्यक अस्थायी नेटवर्क जानकारी सहेजी नहीं जाती
  • फ़ाइल नाम, आकार या टाइमस्टैम्प रिकॉर्ड नहीं किए जाते

4. सेशन कंट्रोल

डाउनलोड लिमिट सेट किया जा सकता है। लिमिट पहुँचने पर सेशन ऑटो डिलीट होता है।

5. एन्क्रिप्शन

सभी ट्रांसफर WebRTC एन्क्रिप्टेड चैनल पर होते हैं।

वैकल्पिक रूप से AES-256 क्लाइंट-साइड एन्क्रिप्शन सक्षम किया जा सकता है।
