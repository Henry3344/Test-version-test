About side

FlashDrop Pro is built with a single principle: your data never belongs to us. This document explains exactly how FlashDrop Pro handles privacy, security, and data protection.

1. Pure Peer-to-Peer Architecture

FlashDrop Pro uses WebRTC-based peer-to-peer connections. Files and text are transferred directly between browsers.

  • No file data is uploaded or stored on servers
  • No cloud storage, no intermediaries
  • Once the browser tab is closed, all session data is destroyed

Signaling data required to establish peer-to-peer connections is temporary, minimal, and never linked to file contents.

2. Zero Server Storage

  • We do not store files or text content
  • We do not store download history or transfer metadata

3. Zero Logs Policy

  • We do not intentionally collect, store, or analyze IP addresses or user identities
  • Any transient network metadata required for connection establishment is not retained
  • No file names, sizes, or timestamps are recorded

4. End-to-End Session Control

Users define strict access limits: one-time download, limited counts, or manual stop. Once reached, session self-destructs automatically.

5. Encryption & Transport Security

All transfers occur over encrypted WebRTC channels using DTLS-SRTP. Data is encrypted in transit and never decrypted by our servers.

Optional client-side AES-256 encryption may be enabled before transmission. Encryption keys are generated and applied locally and are never transmitted or stored.

Transparency Notice

As of January 2026, FlashDrop Pro has never received any government, legal, or third-party requests for user data or content.

关于我们

FlashDrop Pro 建立在一个核心原则之上：你的数据永远不属于我们。本文档详细说明了 FlashDrop Pro 如何处理隐私、安全和数据保护。

1. 纯 P2P 架构

FlashDrop Pro 使用基于 WebRTC 的点对点连接。文件和文本直接在浏览器之间传输。

  • 文件数据绝不会上传或存储在服务器上
  • 没有云存储，没有中间商
  • 一旦浏览器标签页关闭，所有会话数据将被销毁

建立点对点连接所需的信令数据是临时的、最小化的，并且绝不与文件内容相关联。

2. 零服务器存储

  • 我们不存储文件或文本内容
  • 我们不存储下载历史记录或传输元数据

3. 零日志政策

  • 我们不会主动收集、存储或分析 IP 地址或用户身份
  • 建立连接所需的任何瞬态网络元数据都不会被保留
  • 不会记录文件名、大小或时间戳

4. 端到端会话控制

用户定义严格的访问限制：单次下载、限制次数或手动停止。一旦达到限制，会话将自动销毁。

5. 加密与传输安全

所有传输均使用 DTLS-SRTP 在加密的 WebRTC 通道上进行。数据在传输过程中是加密的，服务器永远无法解密。

传输前可启用可选的客户端 AES-256 加密。加密密钥在本地生成和应用，绝不会被传输或存储。

私たちについて

FlashDrop Proは、たった一つの原則に基づいて構築されています。それは「あなたのデータは決して私たちのものではない」ということです。このドキュメントでは、FlashDrop Proがプライバシー、セキュリティ、データ保護をどのように扱っているかを正確に説明します。

1. 純粋なP2Pアーキテクチャ

FlashDrop ProはWebRTCベースのピアツーピア接続を使用します。ファイルとテキストはブラウザ間で直接転送されます。

  • ファイルデータがサーバーにアップロードまたは保存されることはありません
  • クラウドストレージも仲介者も存在しません
  • ブラウザのタブを閉じると、すべてのセッションデータは破棄されます

ピアツーピア接続を確立するために必要なシグナリングデータは一時的かつ最小限であり、ファイルの内容にリンクされることはありません。

2. サーバー保存ゼロ

  • ファイルやテキストの内容は保存しません
  • ダウンロード履歴や転送メタデータは保存しません

3. ログゼロポリシー

  • IPアドレスやユーザーIDを意図的に収集、保存、分析することはありません
  • 接続確立に必要な一時的なネットワークメタデータは保持されません
  • ファイル名、サイズ、タイムスタンプは記録されません

4. エンドツーエンドのセッション制御

ユーザーは厳格なアクセス制限（1回限りのダウンロード、回数制限、手動停止）を定義します。制限に達すると、セッションは自動的に自己破壊します。

5. 暗号化と転送セキュリティ

すべての転送は、DTLS-SRTPを使用した暗号化されたWebRTCチャネル上で行われます。データは転送中に暗号化され、サーバーによって復号されることはありません。

転送前にオプションのクライアント側AES-256暗号化を有効にすることができます。暗号化キーはローカルで生成・適用され、送信または保存されることはありません。

서비스 소개

FlashDrop Pro는 사용자의 데이터는 결코 우리에게 속하지 않는다는 단 하나의 원칙으로 구축되었습니다. 이 문서는 FlashDrop Pro가 개인정보, 보안 및 데이터 보호를 어떻게 처리하는지 정확하게 설명합니다.

1. 순수 P2P 아키텍처

FlashDrop Pro는 WebRTC 기반의 P2P(Peer-to-Peer) 연결을 사용합니다. 파일과 텍스트는 브라우저 간에 직접 전송됩니다.

  • 파일 데이터는 서버에 업로드되거나 저장되지 않습니다.
  • 클라우드 스토리지나 중개자가 없습니다.
  • 브라우저 탭을 닫으면 모든 세션 데이터가 파기됩니다.

P2P 연결을 설정하는 데 필요한 신호 데이터는 일시적이고 최소한이며, 파일 내용과 절대 연결되지 않습니다.

2. 서버 저장소 제로

  • 파일이나 텍스트 콘텐츠를 저장하지 않습니다.
  • 다운로드 기록이나 전송 메타데이터를 저장하지 않습니다.

3. 로그 제로 정책

  • IP 주소나 사용자 신원을 의도적으로 수집, 저장 또는 분석하지 않습니다.
  • 연결 설정에 필요한 일시적인 네트워크 메타데이터는 보관되지 않습니다.
  • 파일 이름, 크기 또는 타임스탬프는 기록되지 않습니다.

4. 엔드투엔드 세션 제어

사용자는 엄격한 액세스 제한(1회 다운로드, 횟수 제한 또는 수동 중지)을 정의합니다. 제한에 도달하면 세션이 자동으로 파기됩니다.

5. 암호화 및 전송 보안

모든 전송은 DTLS-SRTP를 사용하는 암호화된 WebRTC 채널을 통해 이루어집니다. 데이터는 전송 중에 암호화되며 서버에서 절대 복호화되지 않습니다.

전송 전에 선택적 클라이언트 측 AES-256 암호화를 활성화할 수 있습니다. 암호화 키는 로컬에서 생성 및 적용되며 절대 전송되거나 저장되지 않습니다.

Sobre Nosotros

FlashDrop Pro se basa en un único principio: sus datos nunca nos pertenecen. Este documento explica exactamente cómo FlashDrop Pro maneja la privacidad, la seguridad y la protección de datos.

1. Arquitectura P2P Pura

FlashDrop Pro utiliza conexiones peer-to-peer basadas en WebRTC. Los archivos y el texto se transfieren directamente entre navegadores.

  • No se suben ni almacenan datos de archivos en los servidores
  • Sin almacenamiento en la nube, sin intermediarios
  • Una vez cerrada la pestaña del navegador, todos los datos de la sesión se destruyen

Los datos de señalización necesarios para establecer conexiones P2P son temporales, mínimos y nunca están vinculados al contenido de los archivos.

2. Cero Almacenamiento en Servidor

  • No almacenamos archivos ni contenido de texto
  • No almacenamos historial de descargas ni metadatos de transferencia

3. Política de Cero Registros (Logs)

  • No recopilamos, almacenamos ni analizamos intencionalmente direcciones IP o identidades de usuarios
  • No se conserva ninguna metadatos de red transitoria necesaria para el establecimiento de la conexión
  • No se registran nombres de archivos, tamaños ni marcas de tiempo

4. Control de Sesión de Extremo a Extremo

Los usuarios definen límites de acceso estrictos: descarga única, conteo limitado o parada manual. Una vez alcanzado, la sesión se autodestruye automáticamente.

5. Cifrado y Seguridad de Transporte

Todas las transferencias ocurren a través de canales WebRTC cifrados usando DTLS-SRTP. Los datos se cifran en tránsito y nunca son descifrados por nuestros servidores.

Se puede habilitar el cifrado opcional del lado del cliente AES-256 antes de la transmisión. Las claves de cifrado se generan y aplican localmente y nunca se transmiten ni almacenan.

Aviso de Transparencia

A partir de enero de 2026, FlashDrop Pro nunca ha recibido solicitudes gubernamentales, legales o de terceros para datos de usuarios o contenido.

  • सर्वर पर कोई फ़ाइल डेटा अपलोड या स्टोर नहीं किया जाता है
  • कोई क्लाउड स्टोरेज नहीं, कोई बिचौलिए नहीं
  • ब्राउज़र टैब बंद होने के बाद, सभी सत्र डेटा नष्ट हो जाते हैं

पीयर-टू-पीयर कनेक्शन स्थापित करने के लिए आवश्यक सिग्नलिंग डेटा अस्थायी, न्यूनतम है, और फ़ाइल सामग्री से कभी नहीं जुड़ा होता है।

2. शून्य सर्वर स्टोरेज

  • हम फ़ाइलें या टेक्स्ट सामग्री स्टोर नहीं करते हैं
  • हम डाउनलोड इतिहास या ट्रांसफर मेटाडेटा स्टोर नहीं करते हैं

3. शून्य लॉग नीति

  • हम जानबूझकर आईपी पते या उपयोगकर्ता पहचान एकत्र, संग्रहीत या विश्लेषण नहीं करते हैं
  • कनेक्शन स्थापना के लिए आवश्यक कोई भी क्षणिक नेटवर्क मेटाडेटा नहीं रखा जाता है

5. एन्क्रिप्शन और परिवहन सुरक्षा

सभी ट्रांसफर DTLS-SRTP का उपयोग करके एन्क्रिप्टेड WebRTC चैनलों पर होते हैं। डेटा पारगमन में एन्क्रिप्ट किया जाता है और हमारे सर्वर द्वारा कभी भी डिक्रिप्ट नहीं किया जाता है।

ट्रांसमिशन से पहले वैकल्पिक क्लाइंट-साइड AES-256 एन्क्रिप्शन सक्षम किया जा सकता है। एन्क्रिप्शन कुंजी स्थानीय रूप से उत्पन्न और लागू की जाती है और कभी भी प्रेषित या संग्रहीत नहीं की जाती है।
